# Politica di sicurezza

## Segnalare una vulnerabilità

La sicurezza è importante, soprattutto perché la piattaforma raccoglie dati dei giocatori.

**Non aprire una issue pubblica** per le vulnerabilità di sicurezza. Segnalale invece in
modo privato tramite **GitHub Security Advisories** (scheda *Security → Report a vulnerability*
del repository), così da poterle gestire prima della divulgazione.

Nella segnalazione, per quanto possibile, includi:

- una descrizione della vulnerabilità e del suo impatto;
- i passi per riprodurla (proof of concept);
- le versioni/componenti interessati.

## Tempi di risposta

Trattandosi di un progetto open source mantenuto su base volontaria, non sono garantiti tempi
di risposta formali, ma le segnalazioni di sicurezza hanno priorità. Riceverai un riscontro
appena possibile e sarai aggiornato sulla risoluzione.

## Ambito

Rientrano nello scopo le vulnerabilità del codice di questo repository. Le dipendenze di terze
parti vanno segnalate ai rispettivi progetti (gli aggiornamenti automatici sono gestiti via
Dependabot).
